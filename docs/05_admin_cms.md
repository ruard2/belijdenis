# Admin CMS

## Doel

De catecheet moet zonder programmeerkennis inhoud kunnen beheren.

## Hoofdfuncties

### Dashboard

Toont:
- actieve cursussen
- groepen
- recente vragen
- recente ontdekkingen
- concepten die nog niet gepubliceerd zijn

### Cursusbeheer

Admin kan:
- cursus aanmaken
- titel aanpassen
- beschrijving aanpassen
- afbeelding kiezen
- cursus dupliceren
- cursus archiveren
- volgorde wijzigen

### Hoofdstukbeheer

Admin kan:
- hoofdstuk aanmaken
- hoofdstuk dupliceren
- titel en subtitel aanpassen
- XP instellen
- volgorde wijzigen
- draft opslaan
- previewen
- publiceren
- archiveren

### Block editor

Admin kan:
- block toevoegen
- blocktype kiezen
- velden invullen
- block dupliceren
- block verwijderen
- volgorde wijzigen via drag-and-drop
- XP per block instellen
- required aan/uit zetten

### Media

Admin kan:
- afbeelding uploaden
- audio uploaden
- video of link toevoegen
- YouTube-link toevoegen
- Spotify-link toevoegen
- alt-tekst toevoegen
- thumbnail kiezen

### Communitymoderatie

Admin kan:
- ontdekkingen bekijken
- reacties bekijken
- content verbergen
- rapportages bekijken
- vragen vastzetten
- vragen beantwoorden

## UX-principes

- Admin moet weinig technische termen zien.
- JSON mag niet de primaire interface zijn.
- Elk blocktype heeft eigen formulier.
- Preview moet lijken op leerlingweergave.
- Publiceren is een bewuste actie.
- Verwijderen gebruikt bij voorkeur soft delete.

## Minimale admin-schermen

1. Login
2. Dashboard
3. Cursussenlijst
4. Cursus detail
5. Hoofdstuk editor
6. Block editor
7. Preview
8. Groepen
9. Vragen
10. Moderatie
