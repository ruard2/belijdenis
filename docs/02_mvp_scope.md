# MVP Scope

## MVP-doel

Een eerste werkende versie waarmee een catecheet een cursus kan beheren en een groep leerlingen twee seed-hoofdstukken kan laten volgen.

## Must have

- Accountregistratie en login
- Rollen: leerling, catecheet/admin, superadmin
- Cursussen
- Hoofdstukken
- Blocks
- Admin-CMS voor contentbeheer
- Draft/published status
- Preview voor admin
- Groepen met invite code
- Voortgang per gebruiker
- XP-events
- Ontdekkingen bij blocks
- Reacties op ontdekkingen
- Vragen bij blocks
- Antwoorden door catecheet
- Media metadata en upload-flow
- Railway deployment
- Basis testset

## Should have

- Hoofdstuk dupliceren
- Cursus dupliceren
- Blocks drag-and-drop ordenen
- Basic moderation: verbergen, rapporteren
- Badge-toekenning
- Eenvoudige dashboardstatistieken voor catecheet

## Could have

- Streaks
- Pushnotificaties
- Audio-opnames
- Video-upload
- Uitgebreide analytics
- Export van groepsvragen
- Rich text editor

## Won't have in MVP

- Betalingen
- Multi-tenant abonnementen
- AI-contentgenerator
- Volledige offline mode
- Native desktop admin-app
- Complexe ouder/verzorger-workflow

## MVP-releasecriteria

- Backend draait lokaal en op Railway.
- Flutter app kan gepubliceerde content tonen.
- Admin kan content aanpassen zonder codewijziging.
- Seed-hoofdstukken 1 en 2 staan in database.
- Leerling kan blocks afronden en XP ontvangen.
- Leerling kan in eigen groep ontdekkingen en vragen plaatsen.
- Catecheet kan vragen beantwoorden en content beheren.
